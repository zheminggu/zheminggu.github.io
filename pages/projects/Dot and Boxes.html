My Dot and Boxes

This project is an AI of Dot and Boxes, which consists of two parts: user interface (UI) and game engine. The game engine based on Monte Carlo Tree Search (MCTS) is designed by myself. The UI realized with WPS framework using C#.

Dot and Boxes

Rules of Dot and Boxes

Starting with an empty grid of dots, two players take turns adding a single horizontal or vertical line between two unjoined adjacent dots. The player who completes the fourth side of a 1Ã—1 box earns one point and takes another turn. (A point is typically recorded by placing a mark that identifies the player in the box, such as an initial.) The game ends when no more lines can be placed. The winner is the player with the most points. The board may be of any size.

About this project

To download the project, hit the Link here, I've put it on my Google Drive.

To play this game, you should also download the UI

For more details of the code, you can jump to my Github.

To run this project, you should go to the UI folder then run the DotsAndBoxesUI.exe. If it works well, you shall see the User interface. In this project, the board size is 5X5. On the start of a Dot and Boxes, the blue side is always the first hand. The UI will count points and time automatically.

The load Engine button on the right is to load the game engine, the core of this project. The game engine can be found in ../Project1/Release/Project1.exe After load successfully, you can click on the New Game button to start a new game with the computer. To stop the game, hit the Stop Game button on the left side. If you want to play with other players, you can Unlead Engine, start a new game without game engine loaded. The white block on the UI is Information of each action which is designed to debug the game engine.

Game Engine

I created the game engine in the game theory lab's summer training. It was accomplished using C++ with the power of Visual Studio. During the training, I studied some basic algorithm of Game theory, including MinMax, Alpha Beta and Monte Carlo Tree (MCT). With the knowledge I've learned, I designed this Dot and Box game engine with MCT search optimized by UCT.

At first, I want to combine MCTS and Alpha Beta to enhance the prediction ability of the game engine. In my view, MCT can simulate widely, so applying it at the beginning would prevent Alpha Beta to exclude valid action mistakenly. However, in practice, this thought didn't perform well. So I commented out the Alpha Beta part, just using MCT to meet the request of time control.

The source code locates in Project1/Project1. UCTEngine and UCTsearch are MCT part. Evaluate, Moves and Search are Alpha Beta part. Due to I commented out the Alpha Beta, the Search function in Search.cpp is serving for MCT search. Main.cpp runs the game engine, and judge the legal action to be move. For more detailed information about the code, please jump to my Github.
